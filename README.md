## 📌 Overview
**Philosophers** is a 1337 (42 Network) project based on the classic "Dining Philosophers" synchronization problem. The goal is to simulate philosophers sitting at a round table, eating, thinking, and sleeping, without starving.

This project is a deep dive into multithreading, concurrency, and race conditions. For a Software Engineer, mastering synchronization primitives is essential for writing safe, high-performance, concurrent applications.

## 💡 Core Concepts Explored
### Multithreading
**Definition:** Multithreading is a programming and execution model that allows multiple threads to exist within the context of a single process, sharing the process's resources but executing independently.
**Problem Solved:** It solves the problem of blocking execution in CPU-bound or I/O-bound tasks by allowing concurrent execution. In the dining philosophers problem, it allows simulating independent actors that operate simultaneously, highlighting the necessity of synchronization (using mutexes) to prevent race conditions and deadlocks.

## 🚀 Features
- **Threading:** Uses `pthread_create` to represent each philosopher as an independent thread.
- **Mutexes:** Uses `pthread_mutex` to protect shared resources (forks) and prevent race conditions or deadlocks.
- **Precise Timing:** Implements accurate time-tracking using `gettimeofday` to monitor philosopher starvation down to the millisecond.
- **Resource Management:** Ensures clean thread joining and mutex destruction upon simulation end.

## 📥 How to Clone
To clone this project, use the following command:
```bash
git clone git@github.com:Anasqabbal/philosophers.git
cd philosophers
```

## 🛠️ Usage
Compile the simulation:
```bash
make
```
Run the program with the required arguments (the executable `philo` is generated by the Makefile):
```bash
./philo number_of_philosophers time_to_die time_to_eat time_to_sleep [number_of_times_each_philosopher_must_eat]
```
Example (Testing the simulation):
```bash
./philo 5 800 200 200
```
